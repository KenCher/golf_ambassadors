⛳ Golf4us GPS - Course Data Information

📊 How Course Data Works

The Golf4us GPS app uses OpenStreetMap to automatically discover golf courses near you. However, OpenStreetMap has limitations:

  • ✅ Course Location - Accurate GPS coordinates
  • ✅ Course Name - Usually accurate
  • ✅ Number of Holes - Often available
  • ⚠️ Total Par - Sometimes available
  • ❌ Hole-by-Hole Data - Rarely available

⚠️ Why Hole Data May Be Incorrect

Example: Buttonwood Golf Course Hole #1 showing as Par 4 when it's actually a different par.

Reason: OpenStreetMap doesn't have detailed hole-by-hole information (par, yardage, stroke index) for most courses. The app generates estimated hole layouts based on:

  • Standard golf course patterns
  • Typical par distributions (Par 3s, 4s, 5s)
  • Average yardages for each par
  • Standard stroke index patterns

This means: The hole data is an educated guess, not actual course data.

✅ How to Get Accurate Hole Data

Option 1: Use the Course Mapper Tool

  1. Open course-mapper.html on your phone
  2. Walk the golf course
  3. At each hole, mark the tee box and green GPS locations
  4. Enter the correct par, yardage, and stroke index
  5. Export the data as JSON
  6. The app will use your accurate data instead of estimates

Option 2: Manual Course Entry

If you have accurate course data (scorecard, website, etc.), you can manually add it to the app:

{
    id: 'buttonwood-custom',
    name: 'Buttonwood Golf Course',
    location: { lat: XX.XXXX, lon: -XX.XXXX },
    holes: 18,
    par: 72,
    holeData: [
        { number: 1, par: 5, strokeIndex: 1, yardage: 520, ... },
        { number: 2, par: 4, strokeIndex: 7, yardage: 380, ... },
        // ... all 18 holes
    ]
}

Option 3: Community Contribution

Help improve OpenStreetMap by adding detailed golf course data:

  • Visit OpenStreetMap.org
  • Create an account
  • Edit the golf course
  • Add hole-specific tags: hole:1:par=5, hole:1:length=520, etc.

🎯 What's Accurate vs. Estimated

✅ Accurate (from OpenStreetMap):

  • Course GPS location
  • Course name
  • Distance from your location to course
  • Real-time GPS distance calculations
  • Compass direction to green

⚠️ Estimated (generated by app):

  • Individual hole par (unless you map it)
  • Hole yardages (unless you map it)
  • Stroke index (unless you map it)
  • Tee box and green GPS coordinates (unless you map it)

💡 Recommended Workflow

For Casual Use:

The estimated data is good enough for:

  • Finding nearby courses
  • Getting approximate distances
  • General GPS navigation
  • Tracking your position on the course

For Accurate Scoring:

  1. Use the course mapper tool to record accurate hole data
  2. Or manually enter data from the course scorecard
  3. The app will then show correct par, yardage, and stroke index

🔄 Future Improvements

To get more accurate course data automatically, the app could integrate with:

  • Golf Course APIs - Paid services with accurate data (Golfshot, GolfLogix)
  • Course Websites - Scrape scorecard data (with permission)
  • User Contributions - Build a community database
  • USGA Database - Official course ratings (limited hole data)

For now, OpenStreetMap provides free, worldwide course discovery, but hole-specific data requires manual mapping.

Bottom Line: The app accurately finds courses and tracks your GPS location. Hole-specific data (par, yardage, stroke index) is estimated unless you map it yourself using the course mapper tool.
